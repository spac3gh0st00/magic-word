## Security

Both pages are hardened with the following security meta tags:

| Header | Value |
|---|---|
| `Content-Security-Policy` | Whitelists only `self`, Google Fonts, and local media вҖ” blocks XSS and injected scripts |
| `X-Frame-Options` | `DENY` вҖ” prevents clickjacking via iframe embedding |
| `X-Content-Type-Options` | `nosniff` вҖ” blocks MIME sniffing exploits |
| `Referrer-Policy` | `no-referrer` вҖ” nothing leaks when navigating away |
| `Permissions-Policy` | Disables camera, mic, geolocation, and payment APIs |

> GitHub Pages doesn't allow real HTTP header config, so these are implemented as `<meta http-equiv>` tags. For projects on your own domain (Nginx, Apache, Netlify, Cloudflare Pages), see [`SECURITY-HEADERS.md`](./SECURITY-HEADERS.md) for ready-to-paste real header configs вҖ” significantly stronger than meta tags.
